# PDF Watermark

A Chrome extension that adds text watermarks to PDFs entirely in your browser.
**Files never leave your machine.** No upload, no server, no telemetry.

## Features

- Drag-and-drop or click to load a local PDF
- Watermark with English, Chinese (simplified), or mixed text
- Live preview of the first page
- Adjustable parameters: opacity, rotation angle, font size, color
- Three layout presets: single (centered), 2×2 grid, 3×3 grid
- Output PDF preserves the original content; watermark sits on top of every page
- Files up to 100 MB

## Privacy

This extension runs **100% locally**:

- No network requests of any kind
- No data collection or analytics
- No background page or service worker watching your activity
- The `permissions` field in `manifest.json` is empty
- All processing (PDF parsing, font embedding, watermark drawing) happens in
  the popup, in your own Chrome process

## Install (developer mode)

1. Clone this repo
2. `npm install`
3. `npm run build`
4. Open `chrome://extensions/` and enable **Developer mode** (top right)
5. Click **Load unpacked** and select the `dist/` folder

A Chrome Web Store release is planned (see milestone M6 in `plan.md`).

## Usage

1. Click the extension icon in Chrome's toolbar
2. Drop a PDF into the dropzone (or click to pick a file)
3. The first page renders as a preview with the default watermark overlay
4. Adjust:
   - **Watermark text** — supports CJK and ASCII
   - **Layout** — single / 2×2 / 3×3
   - **Opacity** — 0.05 to 1.0
   - **Angle** — −90° to 90°
   - **Font size** — 20 to 120 pt
   - **Color** — any hex color
5. Click **Download watermarked PDF**

The output filename has `_watermarked` appended before `.pdf`.

## Supported text

- **ASCII** — uses Helvetica-Oblique (built into PDF, zero embedding cost)
- **Simplified Chinese** — uses an embedded subset of Noto Sans SC (≈3.35 MB,
  GB2312 Level 1 + Level 2 ≈ 6763 characters plus common punctuation)
- **Mixed** — automatically detected; uses the CJK font

Not supported (would render as `.notdef` glyphs):

- Traditional Chinese (e.g. 臺, 灣) outside GB2312
- Japanese kana / kanji unique to Japanese
- Korean Hangul
- Emoji

These are out of scope for v1. Custom font upload is planned post-launch.

## Development

```bash
npm install        # one-time
npm run dev        # vite dev server (HMR)
npm run build      # production build → dist/
npm test           # unit tests (Vitest)
```

### Regenerating the CJK font subset

The bundled `src/assets/fonts/NotoSansSC-subset.ttf` is generated by:

```bash
# Download the source font (one-time)
mkdir -p /tmp/font-download
curl -sL -o /tmp/font-download/NotoSansSC-Regular.otf \
  "https://github.com/notofonts/noto-cjk/raw/main/Sans/OTF/SimplifiedChinese/NotoSansCJKsc-Regular.otf"

# Regenerate the GB2312 character set + subset
node scripts/gen-gb2312-charset.mjs
node scripts/subset-font.mjs
```

The Noto Sans CJK fonts are licensed under the
[SIL Open Font License 1.1](https://scripts.sil.org/OFL), which permits
redistribution as part of this extension.

### Smoke test

```bash
node scripts/smoke-test.mjs
```

Runs `applyWatermark` against a battery of test PDFs (English, CJK, mixed,
multi-page) and verifies the output renders cleanly via `pdftoppm`.
Requires `poppler` (`brew install poppler` on macOS).
